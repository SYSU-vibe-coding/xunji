# 核心测试用例

## 1. 登录与认证

### TC-LOGIN-01 用户注册并登录成功

- 前置条件：手机号未注册
- 步骤：获取验证码，填写手机号、昵称、密码完成注册；再使用手机号和密码登录
- 预期结果：返回 token，进入首页，用户信息正确

### TC-LOGIN-02 管理员账号密码登录成功

- 前置条件：部署者曾显式设置 `BOOTSTRAP_ADMIN_ENABLED=true` 完成一次管理员初始化，随后已恢复为 `false`
- 步骤：在后台输入管理员账号和密码
- 预期结果：进入管理后台，非管理员账号无法进入

### TC-CERT-01 提交认证申请成功

- 前置条件：用户处于未认证状态
- 步骤：填写学号、姓名并上传证件图，提交申请
- 预期结果：申请记录创建成功，状态变为 `PENDING`

### TC-CERT-02 管理员驳回认证

- 前置条件：存在待审核认证申请
- 步骤：管理员查看申请，选择驳回并填写理由
- 预期结果：用户认证状态变为 `REJECTED`，收到通知

## 2. 发布与搜索

### TC-LOST-01 发布失物成功

- 前置条件：普通用户已登录
- 步骤：填写完整失物信息并提交
- 预期结果：失物记录创建成功，状态为 `SEARCHING`

### TC-FOUND-01 发布招领成功

- 前置条件：拾获者已登录
- 步骤：填写招领信息、设置问题、提交
- 预期结果：招领记录创建成功，状态为 `PENDING`

### TC-SEARCH-01 按类别筛选招领列表

- 前置条件：系统中存在多种类别招领数据
- 步骤：选择指定类别筛选
- 预期结果：仅返回该类别招领记录

## 3. 匹配与认领

### TC-MATCH-01 生成匹配结果

- 前置条件：系统中存在相近的失物和招领记录
- 步骤：发布新招领或触发重算
- 预期结果：生成匹配记录，返回综合得分

### TC-CLAIM-01 一级验证通过

- 前置条件：存在带验证问题的招领记录
- 步骤：用户提交正确答案发起认领
- 预期结果：认领请求创建成功，审核通过后状态更新

### TC-CLAIM-02 认领被拒绝

- 前置条件：存在进行中的认领请求
- 步骤：拾获者选择拒绝并填写原因
- 预期结果：状态变为 `REJECTED`，认领人收到通知

### TC-CLAIM-03 重复认领拦截

- 前置条件：同一用户已存在进行中的认领请求
- 步骤：再次提交相同物品认领
- 预期结果：接口返回重复提交或状态冲突错误

## 4. 交接与结案

### TC-HANDOVER-01 双方确认完成交接

- 前置条件：认领请求已审核通过并创建交接安排
- 步骤：失主确认收取，拾获者确认归还
- 预期结果：认领状态为 `HANDED_OVER`，失物/招领记录同步关闭

### TC-CREDIT-01 交接完成后积分增加

- 前置条件：交接成功
- 步骤：查询双方积分与积分流水
- 预期结果：积分增加且流水记录正确

## 5. 管理后台

### TC-ADMIN-01 审核招领内容

- 前置条件：存在待审核招领记录
- 步骤：管理员审核通过
- 预期结果：记录状态更新，操作日志生成

### TC-REPORT-01 举报处理成功

- 前置条件：用户提交举报
- 步骤：管理员处理举报并给出处置结果
- 预期结果：举报状态变更，必要时扣分或警告成功

## 6. 安全与边界

### TC-SEC-01 未登录访问受保护接口

- 步骤：不携带 token 请求个人中心或发布接口
- 预期结果：返回未登录错误

### TC-SEC-02 普通用户访问管理员接口

- 步骤：普通用户调用后台审核接口
- 预期结果：返回无权限错误

### TC-SEC-03 敏感图片越权访问

- 步骤：普通用户尝试访问原始证件图
- 预期结果：访问被拒绝或返回脱敏资源
